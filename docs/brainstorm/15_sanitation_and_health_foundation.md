# Sanitation and Health Foundation

Purpose: define a concrete sanitation/health loop with map-visible outcomes and clear gameplay impacts.

## 1. Design Intent

Sanitation should:
- be a physical logistics problem, not an invisible number
- produce visible city-state changes (clean vs dirty streets)
- affect settler behavior in understandable ways
- integrate with day/night risk systems

## 2. Core Simulation Loop

1. Waste is generated by houses and selected production buildings.
2. Waste accumulates on source tiles/areas if not collected.
3. Cleaners carry waste to disposal sites.
4. If disposal chain is overloaded, overflow creates dirty streets.
5. Persistent dirt creates concrete penalties.

## 3. Waste Types (Simple Start)

MVP waste classes:
- `household_waste`
- `organic_waste` (food chain byproducts/spoilage)

Optional phase 2:
- `industrial_waste` (smoke/ash byproduct influence)

## 4. Source and Sink Buildings

## 4.1 Sources

- houses (steady small output)
- market/inn/bakery (food-associated output)
- storehouses handling spoilage (additional bursts)

## 4.2 Sinks

- `midden_yard` (basic early disposal)
- `compost_yard` (optional conversion of organic waste to low-value fertilizer item)
- `washhouse` (does not remove waste directly; reduces local dirt impact in radius)

## 5. City-Visible Feedback (No Label Spam)

Map should visually reflect sanitation state:
- clean serviced streets: minimal debris
- stressed streets: scattered refuse props
- overloaded areas: clustered waste piles and dull ground decals

House frontage visuals can react:
- clean frontage on serviced homes
- cluttered frontage where waste is persistent

This naturally communicates neighborhood quality.

## 6. Concrete Gameplay Effects

These effects trigger from persistent local dirt, not single missed pickups.

Settler-level:
- increased chance of "absence" event from assigned work when home area is dirty for multiple nights
- increased chance of hunger interrupt chain when organic waste/spoilage is high nearby

Household-level:
- slower household tier upgrade progress
- downgrade risk if severe filth persists

City-level:
- increases nighttime disorder/theft risk multiplier
- worsens outcomes of selected night events (higher loss bands)

## 7. Timing Rules

Daytime:
- waste generation and collection jobs dominate

Evening:
- collection slows as settlers move to social POIs

Night:
- unresolved dirt contributes to risk resolver in `NightOrderManager`

This links sanitation directly with day/night rhythm.

## 8. Data Model Proposal

Building content additions:
```ts
wasteSource?: {
  wasteType: 'household_waste' | 'organic_waste'
  amountPerDay: number
}

wasteSink?: {
  acceptedWasteTypes: string[]
  capacityPerDay: number
}

washhouse?: {
  radiusTiles: number
  dirtMitigationPerNight: number
}
```

Runtime state:
```ts
sanitationState: {
  dirtByTile: Record<string, number>
  wasteBacklogBySourceId: Record<string, number>
  collectionJobsOpen: number
  collectedToday: number
}
```

## 9. Runtime Integration Map

Potential new manager:
- `packages/game/src/Sanitation/SanitationManager.ts`

Integrations:
- `BuildingsManager`: source/sink metadata
- `SettlerWorkManager`: cleaner jobs
- `StorageManager`: spoilage events feed organic waste
- `PopulationManager`: absence chance hooks
- `NightOrderManager`: sanitation risk contribution

## 10. MVP Scope

Phase 1:
- houses + 2 food buildings generate waste
- one sink building (`midden_yard`)
- one worker role (`cleaner`)
- map dirt visuals
- two gameplay effects:
1. increased absence chance from dirty home areas
2. increased night disorder risk from unresolved dirt

Phase 2:
- washhouse
- compost conversion
- household-tier downgrade behavior from chronic filth

## 11. Tuning and Safeguards

- cap max daily penalty per house
- provide early warning before severe sanitation effects
- make cleanup recovery reasonably fast once player responds
- keep phase 1 readable: few waste types, one sink type, clear feedback
